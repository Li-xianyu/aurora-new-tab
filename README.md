# Aurora New Tab

一个偏工作台风格的浏览器新标签页扩展，聚合了每日 Bing 背景、居中搜索、常用快捷链接、分组快捷标签、浏览历史浏览，以及侧滑设置面板。

这个项目目前面向 Chromium 内核浏览器，重点适配 Chrome 与 Edge。整体实现不依赖构建工具，源码就是可直接加载的扩展目录，便于修改、调试和继续扩展。

## 项目特点

- 新标签页替换为三段式工作台界面：上方浏览历史、中间搜索与时间、下方快捷标签。
- 内置每日 Bing 背景图拉取与本地缓存，尽量减少重复请求。
- 支持百度联想词建议，输入搜索时可以直接上下选择建议项。
- 支持保存当前页面为快捷链接，并按分组管理。
- 支持快捷标签拖拽排序、拖拽跨组移动、右键删除、分组删除。
- 支持根据使用频率在首页中部展示常用快捷链接。
- 支持浏览历史分页加载、时间筛选、全文查找。
- 支持设置面板，包括语言、天气模块、动态效果减弱、快捷链接打开方式。
- 支持扩展命令快捷键，快速保存当前页面。

## 主要界面

### 1. 新标签页主界面

新标签页分为三个垂直区域：

- 上方：浏览历史
- 中间：时间、搜索框、常用快捷链接
- 下方：快捷标签分组列表

滚轮、触摸滑动、部分鼠标拖拽手势都可以在这三个区域之间切换。

### 2. 设置面板

在主界面向左拖拽，可打开设置面板。当前包含两个文字 Tab：

- `通用`
- `快捷链接`

其中可配置：

- 语言切换：`CN / EN`
- 天气模块：开启 / 关闭
- 全局动态效果减弱：开启 / 关闭
- 快捷链接打开方式：`本页跳转 / 新开标签页`

### 3. 保存页面弹窗

点击扩展图标，或使用快捷键后，会打开保存页面弹窗，可完成：

- 编辑标题
- 编辑链接
- 选择分组
- 新建分组
- 删除分组
- 保存当前页面为快捷标签

## 功能详解

### 每日 Bing 背景

- 后台脚本会请求 Bing 每日图片接口。
- 图片会转换为 `data URL` 后缓存到 `chrome.storage.local`。
- 同时尝试下载到本地下载目录 `NewTabDailyBing/`。
- 当天内重复打开新标签页时，优先使用缓存，减少重复请求。

### 搜索

- 默认搜索引擎为 Google。
- 联想词接口使用百度建议服务。
- 支持键盘上下键切换建议项。
- 按回车可直接搜索当前输入或当前高亮建议项。
- 按 `Esc` 可清空输入并收起搜索框。

### 浏览历史

- 默认展示最近历史记录。
- 支持筛选：
  - 全部
  - 今天
  - 昨天
  - 查找
- 在查找模式下，支持进一步时间范围过滤：
  - 全部时间
  - 今天
  - 昨天
  - 7 天内
  - 30 天内
  - 90 天内
- 历史列表使用虚拟渲染和懒加载，避免一次性渲染大量 DOM。

### 快捷标签

- 可以把当前页面保存为快捷标签。
- 每个快捷标签包含：
  - 标题
  - 链接
  - 分组
  - 创建时间
  - 使用次数
  - 最近使用时间
  - 排序索引
- 支持长按拖拽排序。
- 支持跨分组拖拽移动。
- 支持右键删除单个快捷标签。
- 支持删除分组，删除后组内项目会自动回收到默认分组 `Ungrouped / 未分组`。

### 常用快捷链接

- 根据使用次数与最近使用时间计算活跃度。
- 只展示近一段时间内真正常用的链接。
- 展示在主界面中部搜索框下方，方便快速访问。

### 天气模块

- 通过定位获取当前地理位置。
- 使用地理编码接口和天气接口获取当前位置天气。
- 可在设置中直接关闭天气模块。

## 快捷键

当前扩展内置了一个命令快捷键：

- `Alt + S`
  - 作用：打开“保存当前页面”弹窗
  - 说明：仅对可保存的页面有效，通常为 `http / https` 页面

## 交互方式

### 页面切换

- 鼠标滚轮向上：从中间区切换到浏览历史
- 鼠标滚轮向下：从中间区切换到快捷标签
- 触摸滑动也支持相同方向的切换逻辑
- 鼠标拖拽也支持在主界面之间切换

### 设置面板

- 从主界面向左拖拽可打开设置
- 在设置打开状态下向右拖拽可关闭设置

### 搜索框

- 点击搜索按钮展开搜索框
- 输入内容后自动请求联想词
- `ArrowUp / ArrowDown` 切换联想项
- `Enter` 搜索
- `Esc` 关闭并清空

### 快捷标签拖拽

- 鼠标左键长按标签进入拖拽
- 拖拽到其他分组时会自动移动分组
- 松开后自动持久化排序结果

### 分组删除

- 删除分组采用二次确认逻辑
- 被删除分组下的链接不会直接丢失，而是回收到默认分组

## 为什么没有构建步骤

这个项目当前是“源码即运行目录”的结构：

- 没有 `npm install`
- 没有 `npm run build`
- 没有 Webpack / Vite / Rollup

这样做的好处是：

- 结构直接
- 调试简单
- 加载到浏览器里就能运行
- 后续改动后刷新扩展即可验证

## 使用方法

1. 获取本项目源码。
2. 在 Chrome 或 Edge 中自行查找“如何加载未打包的扩展程序”。
3. 选择本项目根目录进行加载。
4. 加载成功后，新建标签页即可看到扩展界面。

说明：

- 这里不展开逐步教学，直接按浏览器自己的“加载未打包扩展”方式处理即可。
- 本项目根目录就是扩展目录，不需要额外构建。

## 权限说明

`manifest.json` 中申请了如下主要权限：

- `storage`
  - 保存用户设置、缓存数据、快捷标签、弹窗状态
- `history`
  - 读取浏览历史
- `tabs`
  - 读取当前活动标签页，并用于打开链接
- `downloads`
  - 下载每日 Bing 图片到本地
- `favicon`
  - 获取站点图标
- `geolocation`
  - 获取当前位置，用于天气显示
- `windows`
  - 获取窗口尺寸，用于保存弹窗定位
- `scripting`
  - 为后续扩展能力预留
- `unlimitedStorage`
  - 为图片缓存与本地数据提供更宽裕的存储空间

## 核心模块说明

### `background.js`

后台服务工作线程，负责：

- 每日 Bing 图片请求、缓存、下载
- 保存快捷标签
- 删除快捷标签 / 分组
- 记录快捷标签使用次数
- 查询历史记录
- 给前端页面提供统一消息接口

### `scripts/main.js`

新标签页主入口，负责：

- 初始化国际化
- 初始化设置面板
- 初始化天气、搜索、历史、快捷标签、背景图
- 同步设置项状态

### `scripts/modules/scroll-screens.js`

负责整个新标签页的滚动、手势和屏幕切换体验，包括：

- 三屏切换
- 设置面板拖拽预览
- 惯性与阈值控制
- Reduced Motion 兼容

### `scripts/modules/history.js`

负责浏览历史区域：

- 调用后台查询历史
- 懒加载和分页
- 虚拟列表渲染
- 标签筛选和搜索

### `scripts/modules/saved-links.js`

负责快捷标签：

- 渲染标签分组
- 右键删除
- 长按拖拽
- 使用统计
- 打开方式控制

### `scripts/save-dialog.js`

负责保存弹窗：

- 读取待保存页面信息
- 处理分组选择
- 新建分组
- 删除分组
- 保存数据并关闭弹窗

## 数据存储

项目主要通过 `chrome.storage.local` 保存以下信息：

- UI 语言
- 天气开关
- 动效减弱开关
- 快捷链接打开方式
- Bing 每日图缓存
- Bing 图片下载状态
- 保存的快捷标签数据
- 保存弹窗临时状态
